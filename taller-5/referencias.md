# Referencias - Taller 5

## Fuentes oficiales verificadas
1. Microsoft Learn. *Microsoft Threat Modeling Tool threats*. Explica STRIDE como parte del Security Development Lifecycle. Disponible en: https://learn.microsoft.com/en-us/azure/security/develop/threat-modeling-tool-threats
2. Microsoft Learn. *Threat modeling for drivers*. Incluye la definición operativa de las seis categorías STRIDE: spoofing, tampering, repudiation, information disclosure, denial of service y elevation of privilege. Disponible en: https://learn.microsoft.com/en-us/windows-hardware/drivers/driversecurity/threat-modeling-for-drivers
3. ISO. *ISO/IEC 27001:2022 - Information security management systems — Requirements*. Página oficial de la norma. Disponible en: https://www.iso.org/standard/27001
4. NIST. *Contingency Planning Guide for Federal Information Systems (SP 800-34 Rev. 1)*. Referencia para disponibilidad, continuidad y recuperación. Disponible en: https://www.nist.gov/publications/contingency-planning-guide-federal-information-systems
5. Consejo Nacional de Acreditación (CNA). *Marco normativo*. Portal oficial con lineamientos y actualizaciones que originan el proceso evaluado. Disponible en: https://www.cna.gov.co/portal/Sistema-Nacional-de-Acreditacion/Marco-normativo/
6. Microsoft Support. *Restore a previous version of a file stored in OneDrive*. Disponible en: https://support.microsoft.com/en-us/office/restore-a-previous-version-of-a-file-stored-in-onedrive-159cad6d-d76e-4981-88ef-de6e96c93893
7. Microsoft Support. *Restore your OneDrive*. Disponible en: https://support.microsoft.com/en-us/office/restore-your-onedrive-fa231298-759d-41cf-bcd0-25ac53eb8a15

## Observación metodológica
Las fuentes oficiales se usaron para sustentar:
- la definición formal de STRIDE;
- la necesidad de controles de disponibilidad y recuperación;
- el contexto normativo del CNA.

La adaptación del caso real fue hecha por el equipo a partir del levantamiento del cliente.
